Architecture Overview

The MATLAB version uses an object-oriented architecture: an abstract GameBase class provides shared infrastructure (scoring, combo system, hit effects, color palette), and each game extends it with 4 abstract methods: onInit, onUpdate, onCleanup, and onKeyPress.

The HTML port mirrors this structure exactly. Each game is a JavaScript class with the same interface. The launcher manages a state machine (menu → active → paused → results) and drives the game loop via requestAnimationFrame.

Frame-Rate Independence

Both MATLAB and HTML versions use the same timing system. Each frame, elapsed real time is measured and physics is scaled proportionally:

// Same formula in both MATLAB and JavaScript
const rawDt = Math.min(now - lastTime, 0.1);  // cap at 100ms
const dtScale = rawDt * 60;                    // reference: 60 FPS

At 60 FPS, dtScale = 1.0. At 30 FPS, dtScale = 2.0. All velocities, gravity, friction, and timers are multiplied by dtScale, ensuring identical game feel regardless of frame rate.

Neon Glow System

The distinctive neon aesthetic is achieved through layered rendering with Canvas shadow properties:

// 3-layer ball rendering (core + glow + aura)
ctx.shadowBlur = 25;
ctx.shadowColor = 'rgba(0, 235, 255, 0.4)';
ctx.fillStyle = 'rgba(0, 235, 255, 0.15)';
ctx.arc(x, y, auraRadius, 0, Math.PI * 2);  // outer aura
ctx.fill();
ctx.shadowBlur = 15;
ctx.fillStyle = 'rgba(0, 235, 255, 0.4)';
ctx.arc(x, y, glowRadius, 0, Math.PI * 2);  // glow ring
ctx.fill();
ctx.shadowBlur = 0;
ctx.fillStyle = '#ffffff';
ctx.arc(x, y, coreRadius, 0, Math.PI * 2);  // bright core
ctx.fill();

Display Scaling

The game uses a fixed logical coordinate system of 854 × 480 (16:9 aspect ratio), matching the MATLAB display range. The canvas scales to fill the browser window while preserving aspect ratio (letterboxed).

For crisp rendering on high-DPI displays, the canvas backing store is sized to cssSize × devicePixelRatio. A coordinate transform maps all drawing calls from 854×480 logical space to the full physical resolution. Text and lines render at native pixel density — no blurriness at any scale.

Shared Systems

Hit Effects

Expanding ring + radial burst rays + floating score text. Two variants ported from GameBase.spawnHitEffect (8 rays, 18 frames) and spawnBounceEffect (5 directional rays, 22 frames). Ease-out animation with per-frame alpha decay and ring expansion.

Combo System

Shared scoring with multiplier: max(1, combo / 10). Auto-fade after 1.0s inactivity, 0.6s fade-out duration. Score roll-up animation at 3 points per frame.

Speed-to-Color Gradient

Multi-segment interpolation: Cyan → Green → Gold → Red at speed thresholds 3, 7, 12. Used across Pong (ball), Flick It (orb), Juggler (ball), and wall bounce effects.

Score Persistence

MATLAB stores scores in ScoreManager_scores.mat. The HTML version uses localStorage with the same per-game record structure: high score, max combo, times played.

What Makes This Port Faithful

Not a reimagination — a translation. Every numeric constant, every collision formula, every scoring rule, every animation curve was extracted from the MATLAB source and reproduced exactly. The games don't just look similar — they behave identically.

Key aspects of fidelity:

  • Same coordinate system (854×480, Y-down)
  • Same physics formulas (parametric collision, swept segments)
  • Same AI behavior (Pong difficulty ramp, alien formations)
  • Same scoring rules (combo multipliers, back-to-back bonuses)
  • Same visual proportions (ball radii, paddle sizes, margins)
  • Same neon color palette (6 primary colors, exact RGB values)
  • Same timing (serve countdowns, invulnerability frames, animation durations)
